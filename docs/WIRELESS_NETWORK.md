# 无线网络与无线话筒设计方案

> 金湖展贸中心智能控制系统 · WiFi 部署 + 无线麦克风选型 + 频段冲突规避
> 最后更新: 2026-05-21

本文档讲两件事:
1. **无线 WiFi** — 控制平板 + 工作人员手机 + 访客 怎么接入
2. **无线话筒** — 演出/路演/接待用的无线麦, 怎么不跟 WiFi 打架

两套系统**物理上独立**, 接收/发射设备不同, 但**频段上可能冲突**, 必须规划好.

## 二、无线 WiFi 设计

### 2.1 接入设备清单

| 类型 | 设备 | 数量 | 接哪个 SSID |
| --- | --- | --- | --- |
| **控制平板** | 1F 主控台 + 2F 副控台 (可选) | 1-2 | `JinHu-Control` (设备网, 能访问 192.168.50.10) |
| **工作人员手机** | 接待/客服/管理员 | 5-10 | `JinHu-Office` (办公网, 上外网, 不能进设备网) |
| **维护笔记本** | 现场调试 | 1-2 | `JinHu-Office` 或 `JinHu-Control` (维护时) |
| **访客手机** | 客户/参观者 | 不限 | `JinHu-Guest` (访客网, 仅出外网) |

### 2.2 SSID 与 VLAN 设计

| SSID | VLAN | 子网 | 网关 | 能访问什么 | 加密 |
| --- | --- | --- | --- | --- | --- |
| `JinHu-Control` (隐藏) | 50 | 192.168.50.0/24 | 无 | 仅设备网 (含主控机) | WPA3, 密码长复杂 |
| `JinHu-Office` | 1 | 192.168.1.0/24 | 路由器 | 设备网 + 外网 | WPA3 |
| `JinHu-Guest` | 100 | 192.168.100.0/24 | 路由器 | **仅外网**, 隔离设备网 | WPA2 (兼容旧设备) |

**为什么平板 SSID 要隐藏**: 防止公众扫描连接尝试爆破. 平板首次配置时手动添加 SSID + 密码, 之后自动连.

### 2.3 AP 选型

| 推荐档次 | 厂家 | 型号 | 单价估 | 覆盖面积 | 特性 |
| --- | --- | --- | --- | --- | --- |
| **A. 推荐** | TP-Link | **CAP1750-PoE** | ¥450-600 | 80-120 m² | WiFi5 双频 · POE · 含 AC 模拟管理 |
| B. 升级 | TP-Link | **EAP670** (WiFi6) | ¥700-900 | 100-150 m² | WiFi6 双频 · AX5400 · POE · 集中管理 |
| C. 工业级 | 华为 eKit | **AirEngine 5760-51** | ¥1,500-2,500 | 120-180 m² | WiFi6 · 企业级 · 高密集 |

**推荐 A (TP-Link CAP1750)**: 性价比之王, 1750Mbps 够用, POE 供电免拉电源线, 支持 VLAN 划分 SSID, 多 AP 漫游.

### 2.4 AP 数量估算 (覆盖)

|  楼层 | 面积 | 建议 AP 数 | 位置 |
| --- | --- | --- | --- |
| 1F | 400-600 m² | **2 个** | 前厅天花 ×1, 展销区天花 ×1 |
| 2F | 300-500 m² | **2 个** | 服务中心 ×1, 共享办公 ×1 |
| 走廊/楼梯 | 共享 | 0 | 用相邻 AP 覆盖 |

**共 4 个 AP**, 总价约 ¥1,800-2,400 (TP-Link CAP1750).

**布点原则**:
- AP 间距 10-15 米左右
- 避免装在金属吊顶下方 (信号被屏蔽)
- 远离 LED 屏 (LED 大屏散热风扇产生 2.4G 噪声)
- 远离微波炉 / 蓝牙音箱密集区

### 2.5 AC 集中管理

多 AP 必须有控制器 (AC) 集中管, 否则每台单独配很痛苦.

| 方案 | 实现 | 成本 |
| --- | --- | --- |
| **方案 A (省钱)** | TP-Link CAP1750 内置 "AP 模式" + 路由器 AC 模拟功能 | 0 (已含) |
| **方案 B (推荐)** | TP-Link **AC500** 独立 AC 控制器 | ¥800-1,200 |
| 方案 C | 云管 (TP-Link Omada / 华为 eKit Cloud) | 0 (免费) |

**推荐 B (AC500)**: 4 个 AP 用云管/无 AC 也能跑, 但配置 SSID/VLAN/漫游/客户端管理时, 有 AC 一站式配置, 后期省时间.

## 三、无线话筒设计

### 3.1 频段选择

**核心原则**: **绝不用 2.4GHz 无线麦**, 会跟 WiFi 互相干扰, 现场动不动断音/啸叫.

| 频段类型 | 是否推荐 | 理由 |
| --- | --- | --- |
| **UHF 470-525 MHz** | ✅ **强烈推荐** | 中国合法 ISM 频段, 跟 WiFi/蓝牙/手机完全错开, 专业演出标准 |
| UHF 615-655 MHz | ⚠ 部分地区可用 | 各地电视广播占用情况不同, 用前查当地 |
| 1785-1805 MHz | ✅ 备选 | 中国合法, 短距离, 适合会议室 |
| 2.4 GHz | ❌ **不要用** | 跟 WiFi 同段, 互相干扰, 不稳 |
| FM 88-108 | ❌ | 已禁用, 不合法 |

> **中国无线话筒合法频段** (工信部 2024 公告): 470-510, 510-525, 565-585, 1785-1805 MHz. 买之前问商家"是不是国标频段", 别买进口未阉割版.

### 3.2 无线麦推荐型号

按预算和场景:

#### 经济型 (¥1,000-3,000 / 一拖二)

| 厂家 | 型号 | 类型 | 频段 | 单价估 | 特性 |
| --- | --- | --- | --- | --- | --- |
| **推荐** | 得胜 TAKSTAR | **TS-7220** | UHF 一拖二手持 | 470-510 MHz | ¥1,200-1,800 | 国产高性价比, 双手持 |
| | JTS | **US-901D-PRO** | UHF 一拖二 | 575-606 MHz | ¥1,500-2,500 | 接待/会议常用 |
| | 索尼 | **UWP-D系列基础款** | UHF 一拖一腰挂 | 双频 | ¥2,500-3,500 | 适合一人主持 |

#### 专业型 (¥3,000-8,000 / 一拖二)

| 厂家 | 型号 | 类型 | 频段 | 单价估 | 特性 |
| --- | --- | --- | --- | --- | --- |
| **推荐** | 舒尔 Shure | **BLX24/PG58** | UHF 一拖二手持 | 470-866 MHz (中国版阉割) | ¥4,500-6,000 | 业界标杆, 音质清晰, 抗干扰 |
| | 森海塞尔 | **EW-D 100 G4** | UHF 一拖二 | 470-558 MHz | ¥5,500-7,000 | 演出级 |
| | 舒尔 | **MX418** (鹅颈) | 有线鹅颈 + 无线腰挂 | 同上 | ¥3,000-4,500 | 会议主席台用 |

**推荐组合**:
- **一拖二手持**: 主持人 + 客户/嘉宾访谈 各一只 (舒尔 BLX24 ¥5,000)
- **腰挂领夹** (可选): 导览/路演移动用 (得胜 TS-9550 ¥800-1,500)
- **鹅颈会议麦** (可选): 接待会议室主席台用 (舒尔 MX418 ¥3,500)

### 3.3 接入 DSP

无线麦接收器输出 **XLR 卡侬线** → DSPPA MAG6816 的 **MIC IN 输入**.

```
[无线手持麦克风]
       ↓ UHF 无线
[UHF 接收器 BLX4 (1U)]
       ↓ XLR 卡侬线 5-10米
[DSPPA MAG6816]
  └─ MIC 1 IN ←┐
  └─ MIC 2 IN ←┘ (一拖二两路独立)
  └─ MUSIC 1 IN (背景音乐)
  └─ ZONE OUT 1-4 → 功放 → 喇叭
```

**DSP 关键设置 (见 [AUDIO_FIELD_INSTALL.md](AUDIO_FIELD_INSTALL.md))**:
- MIC 通道开 **+48V 幻象电源** (动圈麦不需, 电容麦需)
- 开 **Auto Ducking** (MIC 一开, 背景音自动压低 12-18dB)
- 设 **EQ 限幅** (防尖锐/啸叫)
- **MIC 优先级** > 背景音 > 系统提示音

### 3.4 接收器安装位置

固定装在 **1F 主控室机柜** (DSP 旁边):
- 接收器是 1U 标准机架式, 装在机柜 DSP 上方/下方
- 天线 (BNC 母头) 用 **延长线** 拉到机柜外侧或顶部 (机柜屏蔽信号)
- 不能塞在机柜底层 (信号弱)
- 不能挨着 WiFi AP / LED 控制器 (干扰)

## 四、频段冲突规避

### 4.1 频段速查表

```
─────────────────────────────────────────────────────────────
470 ─┬─ UHF 无线话筒 (470-525 MHz)
     │
525 ─┘
     │  ← 空闲 / 电视广播 (各地占用不同)
1785 ┬─ UHF 无线话筒 (1785-1805 MHz)
     │
1805 ┘
     │  ← 移动通信 4G/5G
2400 ┬─ WiFi 2.4G (信道 1/6/11)
     ├─ 蓝牙 (跳频)
2483 ┘  ← 别用 2.4G 麦克风!
     │
5170 ┬─ WiFi 5G (信道 36-64)
     │  (室内, 不需要 DFS)
5330 ┘
     │
5490 ┬─ WiFi 5G (信道 100-144)
     │  (DFS 信道, 需检测雷达)
5730 ┘
     │
5735 ┬─ WiFi 5G (信道 149-165)
5835 ┘  (室外可用)
─────────────────────────────────────────────────────────────
```

### 4.2 WiFi 信道规划

**2.4G** 只用 **1 / 6 / 11**:
- AP-1F-1 → 信道 1
- AP-1F-2 → 信道 6
- AP-2F-1 → 信道 11
- AP-2F-2 → 信道 1 (复用, 跟 AP-1F-1 距离够远)

**5G** 信道更多, AC 自动分配即可.

### 4.3 现场踩点检查 (开馆前必做)

工程师带笔记本 + WiFi 分析 App (iPhone WiFi Explorer / Android WiFi Analyzer) 到展厅:
1. **扫一遍 2.4G 信道占用**, 看周边商户哪些信道密, 自己挑空的
2. **扫一遍 5G 信道**, 优先选 36/40/44/48 (低信道, 室内)
3. **走一遍** 1F 和 2F 全区, 看信号强度 (RSSI > -65 dBm 才算好)
4. **测吞吐**, 用 iPerf3 测速度 (期望 > 100 Mbps 单设备)

### 4.4 干扰源黑名单 (远离这些)

| 干扰源 | 频段 | 应对 |
| --- | --- | --- |
| LED 大屏 (HUB75 信号) | 宽频杂散 | AP 不能装在屏体附近 5 米内 |
| LED 电源开关 | 杂散 | AP 远离强电箱 |
| 微波炉 (员工厨房) | 2.4G | AP 不在厨房隔壁 |
| 蓝牙音箱 | 2.4G | 演出时关掉 |
| 手机热点 | 2.4G | 工作人员开热点导致信道污染 |
| 邻铺 WiFi | 2.4G | 用 5G 优先, 物理隔离 |

## 六、配置步骤 (新设备到货后)

### 6.1 路由器 (TL-ER605) 配 VLAN

1. Web 登录路由器 (默认 192.168.1.1)
2. 接口设置 → 创建 3 个 VLAN:
   - VLAN 1: 192.168.1.0/24 (办公)
   - VLAN 50: 192.168.50.0/24 (设备)
   - VLAN 100: 192.168.100.0/24 (访客)
3. DHCP 仅在 VLAN 1 和 VLAN 100 开启 (VLAN 50 设备网建议静态)
4. 防火墙规则:
   - VLAN 100 (访客) 禁止访问 192.168.50.0/24 和 192.168.1.0/24
   - VLAN 50 (设备) 限制只能访问外网 80/443 (CDN, 升级用)

### 6.2 交换机 (S1224R) 配 Trunk

1. Web 登录交换机
2. 创建 VLAN 1 / 50 / 100
3. 端口 1-2 (主控机) Access VLAN 50 (NIC1), Access VLAN 1 (NIC2)
4. 端口 3-10 (弱电网关) Access VLAN 50
5. 端口 11-14 (AP) **Trunk**, 允许 VLAN 1, 50, 100
6. 端口 23 (上联路由器) Trunk

### 6.3 AP (CAP1750) 配 3 SSID

通过 AC500 控制器或路由器内置 AC:
1. 创建 3 个 SSID:
   - SSID `JinHu-Control` → 关联 VLAN 50, WPA3, 密码长且复杂, **隐藏 SSID**
   - SSID `JinHu-Office` → 关联 VLAN 1, WPA3
   - SSID `JinHu-Guest` → 关联 VLAN 100, WPA2 (兼容旧设备), 限速 5Mbps
2. 漫游: 开启 802.11k/v/r 快速漫游
3. 信道: 2.4G 自动分配 1/6/11, 5G 自动
4. 功率: 室内 50% 即可, 满功率反而干扰多

### 6.4 平板首次配 WiFi

1. 平板设置 → WiFi → **添加网络**
2. SSID: `JinHu-Control` (手动输入)
3. 加密: WPA3 Personal
4. 密码: 现场约定
5. 勾选 "自动连接"
6. 浏览器测试: 打开 `http://192.168.50.10/control/` (内网) 和 `https://cnjinhu.top/control/` (外网) 都应正常

### 6.5 无线麦克风对频

1. 接收器接 220V 电源 + XLR 输出接 DSP MIC IN
2. 接收器面板按 SET → 自动扫频 (找空闲信道)
3. 手持麦克风按对应按钮 → 接收器 → 完成对频
4. 试音, DSP 音量调到 -6dB 不削波
5. 多套同时用必须**各占不同信道**, 间隔 ≥ 250kHz

- [ ] 手机连 `JinHu-Guest`, 访问外网正常

## 八、常见问题

**Q: 平板时不时连不上 PWA, 但 WiFi 显示连着?**
A: 检查 AP 关联的 VLAN 是不是 50; 不是的话平板访问不到 192.168.50.10. 另用 `nslookup 192.168.50.10` 在平板浏览器开发工具看 DNS.

**Q: 无线麦克风串台 (听到别人讲话)?**
A: 跟邻铺/楼上的同频段无线麦冲突. 进接收器菜单, 手动改信道 (前后差 250kHz 以上).

**Q: 演出时无线麦克风突然爆音/啸叫?**
A: 1) MIC 太近喇叭, 距离至少 3 米; 2) DSP 的 EQ 没设防啸叫; 3) 电池快没电了, 备一组镍氢电池现场换.

**Q: 访客 WiFi 被滥用看视频, 占带宽?**
A: 路由器配 `JinHu-Guest` 限速 5Mbps + 限会话数. 极端情况开 captive portal 让访客先关注公众号.

**Q: 双频 WiFi 平板自动连了 2.4G 反而慢?**
A: SSID 同名时, 让 AC 开 "5G Priority" 模式, 优先把双频终端推到 5G.

**Q: 平板上的 PWA Service Worker 缓存有问题, 老看到旧版?**
A: 平板 Chrome → 设置 → 隐私 → 清除缓存 (仅站点数据), 或者强制刷新 (按住刷新按钮).

## 九、安全提醒

1. **JinHu-Control 密码绝不公开**, 只装到平板上, 不告诉访客/员工
2. **定期换密码**, 半年一次, 员工离职立即换
3. **访客网限速**, 防止滥用拖垮主网
4. **设备网封禁外网**, 防勒索病毒经平板入侵
5. **AC 控制器** Web 管理端口仅管理网可访问 (路由器 ACL)
6. **关闭 WPS** (一键配对协议, 有已知漏洞)
7. **关闭 SSID 广播** 对 JinHu-Control (低门槛防护)
8. **MAC 白名单** (可选, 极端要求时启用, 只允许指定平板 MAC 接入)
