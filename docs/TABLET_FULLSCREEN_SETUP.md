# 平板 PWA 全屏使用指南

> 现场操作员手册 · 2026-05-25
> 解决: 浏览器有地址栏遮挡 / 屏幕滑动 / 不像原生 App

---

## 问题: 为什么默认不是全屏?

直接在浏览器 (Safari / Chrome) 里访问 `http://192.168.124.10:5173/control/` 时:
- 顶部有**浏览器地址栏 + 标签页**, 吃掉 60-100px 高度
- 底部有 **iOS Home Bar / Android 导航条**
- 内容区被压缩 → 出现滚动条 / 切页时抖动

**解决方法**: 把 PWA "添加到主屏幕", 之后从主屏图标启动就是**全屏沉浸式**, 跟原生 App 一样。

---

## iPad / iPhone Safari (iOS)

1. 用 **Safari** 打开: `http://192.168.124.10:5173/control/`
   (Chrome 不支持添加 PWA 到主屏, 必须用 Safari)
2. 点底部 (或右上, 看 iPad 位置) **分享按钮** (□↑)
3. 滑动菜单 → 点 **添加到主屏幕**
4. 名字保留 "金湖中控" → 点 **添加**
5. 退 Safari, 在 **主屏幕找到 "金湖中控" 图标**
6. **点图标启动** → 全屏 + 横屏锁定 + 无地址栏

---

## Android 平板 Chrome

1. 用 **Chrome** 打开: `http://192.168.124.10:5173/control/`
2. 右上角 ⋮ 菜单 → **添加到主屏幕** (有的版本叫 "安装应用")
3. 弹窗确认 → 名字保留 "金湖中控"
4. 主屏出现图标 → 点启动 → 全屏

---

## 用 GK9000 自己当大屏 (主控室触屏)

GK9000 旁边接的触摸屏直接用 Chrome 浏览器 **Kiosk 模式** 启动, 自动全屏 + 锁定:

### 方法 A: 桌面快捷方式
1. 桌面右键 → 新建 → 快捷方式
2. 位置填:
   ```
   "C:\Program Files\Google\Chrome\Application\chrome.exe" --kiosk --app=http://localhost:5173/control/ --start-fullscreen --noerrdialogs --disable-translate --no-first-run --fast --fast-start --disable-pinch
   ```
3. 名字 "金湖中控 (Kiosk)"
4. 双击启动 → 全屏 + 隐藏所有 UI + 锁右键

### 方法 B: 开机自启 (无人值守)
把上面的快捷方式**拷到启动文件夹**:
- 按 `Win+R` → 输 `shell:startup` → 回车
- 把快捷方式拖进去
- 重启 GK9000 → 自动登录 → 自动启动 Chrome Kiosk → 直接显示 PWA

退出 Kiosk: 按 `Alt+F4` (有键盘) 或 `Ctrl+Shift+W` (有触屏键盘).

## 验证 PWA 真的全屏了

启动后看一下:
- ✅ **没有浏览器地址栏**
- ✅ **没有标签页 / 收藏夹栏**
- ✅ **横屏自动锁定** (iOS Safari 添加到主屏后)
- ✅ 状态栏 (顶部时间) **隐藏** (iOS) 或仅黑色窄条 (Android)

如果看到任意一项**仍然在**, 说明还在浏览器里跑, 不是 PWA 模式. 重做"添加到主屏"步骤.

## 常见问题

### Q: 添加到主屏后图标灰色 / 打不开?
A: 网络断了或后端 GK9000 关机. PWA 缓存了 HTML/JS, 但 API 调用要后端在线. 确认 GK9000 通电 + 网线插着.

### Q: 全屏后操作不灵敏 / 卡顿?
A: PWA 走 WiFi 比有线慢. 平板尽量靠 AP. 或者用有线投屏接到 GK9000 那块大屏.

### Q: iPad Safari 横屏锁不住, 旋转跟着转?
A: 这是 iOS 16+ 的限制, "添加到主屏"后默认横屏锁定, 但 iOS 16.4+ 用户可在控制中心强制旋转. 想完全锁横屏, 用 iPad 的 **"引导式访问"** (设置 → 辅助功能 → 引导式访问).

### Q: PWA 启动后报 "无法连接服务器"?
A: 你的平板可能不在 192.168.124.x 同一网段. 确认平板连的是办公 WiFi (跟 GK9000 同 AP).
