# 音响系统架构设计

> 金湖展贸中心智能控制系统 · 音响子系统终极方案
> 三目标: LED 大屏高品质音效 / 讲解员跟随式无线麦 / 会议室独立控制
> 最后更新: 2026-05-21

---

## 0. 三目标拆解

| # | 目标 | 关键难点 | 解决方案 |
| --- | --- | --- | --- |
| 1 | LED 主大屏区高品质音效 | 视频音轨有低频, 普通吸顶喇叭撑不起 | 全频主音箱 + 低音炮 + 大功率功放, **独立 1 路 DSP 输出** |
| 2 | 讲解员移动 → 当前区域响应 | 真正"定位+自动路由"太贵, 误判风险高 | **PWA 上"讲解区跟随"按钮** 软切换, 讲解员/助手手动点 |
| 3 | 会议室独立 | 不能被讲解员路由"波及" | DSP 中**会议室通道不接到任何 MIC 路由表** |

## 1. 总体架构

```
[GK9000 主控机]
    ├─ USB 声卡 / HDMI 抽音器 → DSP MUSIC IN (背景音 + 视频音轨)
    ↓ 网络控制 (TCP/HTTP)
    │
[DSP 数字音频矩阵: DSPPA MAG6864 (8 路输出)]
    │
    │ ┌── INPUTS ──────────────────────────────┐
    │ │ MIC 1: 讲解员主用手持麦 (UHF 无线)      │
    │ │ MIC 2: 讲解员备用 / 路演主持            │
    │ │ MIC 3: 会议室鹅颈麦 (有线)              │
    │ │ MIC 4: 会议室手持麦 (UHF 无线)          │
    │ │ MUSIC 1: 背景音乐 (GK9000 → DSP)        │
    │ │ MUSIC 2: 视频音轨 (LED 大屏内容)        │
    │ └─────────────────────────────────────────┘
    │
    │ ┌── 矩阵路由 (动态可调) ───────────────────┐
    │ │ 每个 INPUT 可路由到任意 OUTPUT, 独立增益  │
    │ │ 后端通过 MAG6864 网络协议改路由+音量      │
    │ └─────────────────────────────────────────┘
    │
    └─ OUTPUTS (8 路独立功放):
        ├─ ZONE 1: ★ 1F LED 大屏区  → 大功率功放 + 全频音箱×2 + 低音炮 ⭐⭐⭐⭐⭐
        ├─ ZONE 2:   1F 路演发布洽谈区 → 中功率 + 吸顶喇叭×4 ⭐⭐⭐
        ├─ ZONE 3:   1F 前厅+走廊      → 小功率 + 吸顶喇叭×6 ⭐⭐
        ├─ ZONE 4:   1F 展位群 (3 企业展位+直播间+外贸+综合) → 吸顶喇叭×6 ⭐⭐
        ├─ ZONE 5:   2F 前厅+服务中心  → 吸顶喇叭×4 ⭐⭐
        ├─ ZONE 6:   2F 商务办公区 (集团/决策/研究/指挥) → 吸顶喇叭×4 ⭐⭐
        ├─ ZONE 7:   2F 共享办公       → 吸顶喇叭×4 ⭐⭐
        └─ ZONE 8:   ★ 2F 会议室       → 独立功放 + 高品质会议音箱 ⭐⭐⭐⭐
```

### 关键差异 (vs 原 4 路 MAG6816 方案)

| 项 | 原方案 | 新方案 |
| --- | --- | --- |
| DSP | DSPPA MAG6816 (4 路) | **DSPPA MAG6864 (8 路)** |
| MIC 输入 | 2 路 | **4 路** (含会议室独立 MIC) |
| 输出分区 | 4 (1F 主/2F 主/会议/路演) | **8** (LED 单独 1 路, 会议室单独 1 路, 6 路覆盖剩余区域) |
| 功放 | 4 台同档 | **1 台大功率 (LED) + 1 台高品质 (会议) + 6 台中小功率** |
| 主大屏音响 | 跟其他区一起小音箱 | **专用全频音箱+低音炮** |
| 会议室隔离 | 跟 1F/2F 主区共线 | **物理独立通道** |

## 2. 目标 ① LED 大屏区 — 高品质音效详解

LED 大屏区是**主要表演场**: 播放企业宣传片 / 路演视频 / 节假日内容, 必须有"影院级"音效.

### 设备选型

| 设备 | 推荐型号 | 数量 | 单价估 | 备注 |
| --- | --- | --- | --- | --- |
| 全频主音箱 | **JBL CONTROL 28-1** 或 **DSPPA DSP-280** 8" 2 分频 | 2 | ¥1,200-1,800/只 | 左右各 1 只, 落地或吊装 |
| 低音炮 | **JBL CSS-SUB12** 12" 主动低音炮 或 **DSPPA SUB-1500** | 1 | ¥2,500-4,000 | 桌下/侧面安放 |
| 功放 | **DSPPA MP-8500U** 500W 立体声 (跟 MAG6864 同厂家) | 1 | ¥1,500-2,200 | 接 ZONE 1 输出, 推 2 只主音箱 |
| 喇叭线 | RVS 2×2.5mm² | 30m | ¥6 | 短距粗线, 减少损耗 |
| **小计** | | | | **¥6,400-10,200** |

### 音轨获取

GK9000 上播放视频时, **音频跟视频走 HDMI**:
- GK9000 HDMI 1 → VX1000 → LED 屏 (LED 屏一般**不出声**)
- 需要**单独抽音**: GK9000 → DSP MUSIC IN

三种抽音方案:

| 方案 | 设备 | 优劣 |
| --- | --- | --- |
| **A (推荐)**: GK9000 USB 声卡输出 | 创新 Sound Blaster X3 / 福克斯 Scarlett 2i2 | 软件友好, 音质好, ¥300-800 |
| B: HDMI 音频抽取器 | HDMI Audio Extractor 4K | HDMI → HDMI + 3.5mm + 光纤. 中等. ¥150-400 |
| C: GK9000 主板自带音频 | 3.5mm 输出 | 最简单, 但通常底噪较大 |

**推荐方案 A** — USB 外置声卡, 接 DSP 的 MUSIC IN. 实测信噪比最好.

## 3. 目标 ② 讲解员跟随式无线麦 详解

### 业务场景

讲解员 (1-2 人) 带 UHF 无线话筒, 穿过不同展区:
1. 走到 "1F 路演发布区" 介绍项目, 该区音响放大他声音
2. 走到 "1F 企业展位群" 介绍企业, 该区音响接管
3. 走到 "2F 投资决策中心" 给客户讲, 该区接管

**核心**: 讲解员到哪个区, 哪个区的喇叭就放他的声音, **其他区不打扰** (或保持背景音乐).

### 技术方案对比

| 方案 | 实现 | 设备成本 | 体验 | 推荐 |
| --- | --- | --- | --- | --- |
| **A. PWA 软切换 (推荐)** | 讲解员/助手用手机访问 `/control/#/audio/follow`, 点"我在 X 区"按钮, 后端调 DSP 改路由 | 0 (复用我们 PWA) | 一键切换, 0.5-1s 延迟 | ⭐⭐⭐⭐⭐ |
| B. NFC 标签触发 | 各区门口贴 NFC, 手机靠近触发 API | NFC 标签 ¥10×14 + 手机已有 | 体验更好, 但要讲解员主动靠近 | ⭐⭐⭐⭐ (后期升级) |
| C. 蓝牙信标 (BLE) | 各区放 BLE Beacon, 手机扫描自动切 | Beacon ¥50-150×14 + APP 开发 | 自动化高, 但漂移大 | ⭐⭐ (复杂) |
| D. UWB 厘米级定位 | 各区装 UWB 基站, 讲解员带 UWB 标签 | ¥30,000+ | 真自动, 室内定位精度 10cm | ⭐ (太贵) |
| E. 阵列麦 AI 追踪 | 顶部装麦克风阵列 (Shure MXA920 等) | ¥80,000+ | 完全无感, 但单点失效 | ⭐ (顶级) |

### 方案 A 实现方式

**前端 (`/control/#/audio/follow` 平板新页面)**:
```
┌──────────────────────────────────────┐
│ 🎤 讲解员跟随                         │
│ 当前讲解区: 1F 路演发布洽谈区        │
├──────────────────────────────────────┤
│ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐    │
│ │1F   │ │1F   │ │1F   │ │1F   │    │
│ │前厅 │ │路演*│ │展位 │ │交易 │    │
│ └─────┘ └─────┘ └─────┘ └─────┘    │
│ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐    │
│ │2F   │ │2F   │ │2F   │ │2F   │    │
│ │前厅 │ │决策 │ │研究 │ │指挥 │    │
│ └─────┘ └─────┘ └─────┘ └─────┘    │
├──────────────────────────────────────┤
│ [🔇 退出讲解 (关麦)]                  │
│ [⚙️ 麦克风音量: ━━━●━━ 60%]          │
└──────────────────────────────────────┘
```

讲解员或助手手持平板/手机, 一目了然.

**后端 API**:
```typescript
POST /api/audio/guide/follow
  body: { zoneCode: 'roadshow' }
  // 内部调 DSP: ZONE_2 的 MIC 1 增益打开, 其他 ZONE 的 MIC 1 静音
  // 音量保持当前

POST /api/audio/guide/leave
  // 所有 ZONE 的 MIC 1 静音
```

**DSP 矩阵命令**:
```
切到"路演区":
  MIC 1 → ZONE 2 (路演) 增益 0dB
  MIC 1 → ZONE 1 (LED 大屏) 增益 -∞ (静音)
  MIC 1 → ZONE 3-7 增益 -∞
  MIC 1 → ZONE 8 (会议室) **永远 -∞** (硬规则)
```

**Auto Ducking**: MIC 1 一开, 该 ZONE 的背景音 (MUSIC 1) 自动压低 12dB (DSPPA 内置功能).

### 后期升级路径 (方案 B NFC)

后期想升级到 NFC 自动切, 不需要换硬件:
1. 各区门口贴 NFC 标签 (写入 zone code)
2. 讲解员手机装一个 PWA 快捷, 扫 NFC → 调同一个 API
3. 兼容 PWA 软切换 (两种方式并存)

## 4. 目标 ③ 会议室独立 详解

### 设计原则

会议室 = **音响隔离区**:
- **不接到任何讲解员 MIC 路由** (硬规则)
- 独立 DSP 输出通道 (ZONE 8)
- 独立功放
- 独立 MIC (鹅颈+手持各一支)
- 独立音源 (可接笔记本 HDMI 音轨 / 蓝牙 / 本地音乐)

### 设备清单 (会议室专用)

| 设备 | 推荐型号 | 数量 | 单价估 |
| --- | --- | --- | --- |
| 高品质会议音箱 | **JBL Control 25-1** 5.25" 吸顶 | 4 | ¥800/只 |
| 会议功放 | **DSPPA MP-200U** 200W | 1 | ¥800 |
| 鹅颈会议麦 | **舒尔 MX418** | 2 | ¥3,500/支 (装主席台) |
| 手持无线麦 | **舒尔 BLX24/PG58** 一拖二 | 1 套 | ¥5,000 |
| HDMI 抽音器 (笔记本接入) | 通用 4K HDMI Extractor | 1 | ¥300 |
| **小计** | | | **¥17,400** |

### 会议室控制 (跟主控解耦)

会议室可以**独立平板控制**, 不被主控台干扰:
- 会议室门口装一台平板 (10")
- 平板访问 `/control/#/meeting`
- 仅控会议室设备: 音响开关 / 音量 / MIC / 灯光 / 投影
- 主控台 (1F 前厅) 看得到会议室状态, 但**不轻易干涉**

export const AUDIO_ZONES: AudioZoneConfig[] = [

## 8. 场景动作集成

我们 6 个场景里, 跟新音响相关的动作:

- **opening 开馆**: ZONE 1-7 播背景音乐, ZONE 8 (会议室) 静默
- **roadshow 路演**: ZONE 1 (LED) 视频音轨大音量, ZONE 2 (路演) MIC 路由开
- **meeting 会议**: ZONE 8 独立, MIC 3+4 开, 其他区不动
- **reception 接待**: ZONE 5 (2F 前厅) 柔和背景音
- **cleaning/closing**: 全部静默 / 静音

后续可以**自动在场景里加 audio-zone 类型动作**, 跟 hvac-zone 类似的"扇出"模式.
