**关键设计点**:

- **两台网关物理独立**: 一台外机一台网关, 故障隔离 (1F 网关挂不影响 2F)
- **网关走 RJ45 + Modbus TCP**: 跟主控同一交换机, 不需要 USR-TCP232 转换
- **XYE 总线**: 中弘网关跟奥克斯外机用 RVVP 3 线连, 屏蔽层只**一端接地** (外机端)
- **网关 IP 配置**: 中弘网关默认 DHCP, 出厂上电后需固定 IP, 见 §4

## 4. 中弘网关接线 + 配置

### 4.1 端子定义 (网关背板)

| 端子 | 含义 | 接线 |
| --- | --- | --- |
| 12V+ / 12V- (或 GND) | DC 12V 输入 | 配 DC 12V/1A 适配器, 或弱电箱内 12V 电源 |
| X / Y / E | XYE 通讯总线 (接奥克斯外机) | RVVP 3×0.75 |
| LAN (RJ45) | 以太网口 | Cat5e 接弱电交换机 |
| 旋转拨码 | SlaveID (0-15) | 默认 1, 多网关并 RS485 总线时区分 (本项目 TCP 不需要) |
| RST | 复位按钮 | 不要随便按 (出厂复位 IP) |

### 4.2 接线步骤 (1F 网关示例, 2F 同)

1. **断电**: 关闭 12V 适配器电源
2. **接 XYE 线** (4 米内, 控制柜到外机):
   - 网关 X → 1F 外机 X
   - 网关 Y → 1F 外机 Y
   - 网关 E → 1F 外机 E (屏蔽地, 只外机端接地)
3. **接电源**: 12V 适配器接 12V+/12V-
4. **接网线**: RJ45 → 弱电交换机 (任意空口)
5. **上电**: 工作灯亮即正常

### 4.3 IP 配置 (关键!)

中弘网关默认 DHCP, 要固定到 **192.168.50.51 (1F) / .52 (2F)**.

**方法 A — Web 页面配置 (推荐)**:

1. 笔记本网卡临时改到 DHCP, 接上同交换机
2. 在路由器 DHCP 列表里找到网关 MAC (中弘 OUI 前缀通常是 `00:0E:E0` 或 `E0:E1:E2`)
3. 浏览器访问该 IP (如 192.168.1.100), 进入网关 Web 页 (默认账号 `admin / admin`)
4. 网络配置 → **静态 IP**:
   - 1F 网关: `192.168.50.51 / 255.255.255.0 / 192.168.50.1`
   - 2F 网关: `192.168.50.52 / 255.255.255.0 / 192.168.50.1`
5. 协议选择: **MODBUS-TCP (MBT)**
6. SlaveID: **1** (默认即可)
7. 保存, **重启网关**生效

**方法 B — ARP 临时联通**:

```powershell
# Windows 主机
arp -s 192.168.50.51 00-0e-e0-xx-xx-xx
ping 192.168.50.51
# 然后浏览器 http://192.168.50.51 配置
```

### 4.4 网关搜索内机

网关上电 + 通讯线接好后, **每 3 秒**主动给奥克斯外机发性能查询帧, 直到识别到所有内机. 需要等待 1-2 分钟.

确认搜索完成: 用 Modbus Poll 读寄存器 2000-2005:
- 2000: 品牌 — 应等于 **15 (奥克斯)**
- 2001: 产品类型 — 应等于 **2 (集控网关)**
- 2002: 内机数量 — 应等于 **10 (1F) / 12 (2F)**
- 2003-2004: 温度上下限 (16-32℃)

## 5. Modbus Poll 寄存器调试

> **必做!** 在跟中控联调之前, 先用 Modbus Poll 工具直接读写寄存器, 确认协议是否对.

### 5.1 Modbus Poll 连接

1. File → New
2. Connection → Connect:
   - Connection: **Modbus TCP/IP**
   - IP: `192.168.50.51` (1F) 或 `192.168.50.52` (2F)
   - Port: `502`
   - Slave ID: `1`
3. Slave Definition:
   - Function: **03 Read Holding Registers**
   - Address: `2000` (网关性能信息)
   - Quantity: `6`
   - Scan Rate: `1000 ms`

### 5.2 测试用例

| 测试 | 操作 | 预期 |
| --- | --- | --- |
| 读网关品牌+内机数 | FC03, 2000, 6 寄存器 | 2000=15, 2001=2, 2002=10 (1F) 或 12 (2F) |
| 读 n=0 内机状态 | FC03, 0, 6 寄存器 | 全 0 = 关机; 非全 0 = 内机存在并已上报 |
| 开 n=0 | FC06, 4000, 值 1 | 内机风机启动 (3-5 秒延迟) |
| 关 n=0 | FC06, 4000, 值 0 | 内机风机停 |
| 设制冷 | FC06, 4001, 值 2 | 内机模式切到制冷 |
| 设制热 | FC06, 4001, 值 1 | 切到制热 |
| 设送风 | FC06, 4001, 值 4 | 切送风 |
| 设除湿 | FC06, 4001, 值 8 | 切除湿 |
| 设温度 25℃ | FC06, 4002, 值 0x19 (25) | 内机线控器显示 25℃ |
| 设温度 20℃ | FC06, 4002, 值 0x14 (20) | 显示 20℃ |
| 设高速 | FC06, 4003, 值 3 | 风机高速 |
| 设中速 | FC06, 4003, 值 2 | 中速 |
| 设低速 | FC06, 4003, 值 1 | 低速 |
| 设自动风 | FC06, 4003, 值 0 | 自动 |
| 一次写全部参数 (开+25℃+制冷+高风) | FC10, 4000, 4 寄存器 [1, 2, 0x19, 3] | 一帧搞定 |
| 全部内机开机 | FC06, 5000, 值 1 | 该网关所有内机同步开 |

### 5.3 风速编码 (⚠ 注意!)

| 命令风速 | 中弘 B 集控值 | 跟其他协议对比 |
| --- | --- | --- |
| 自动 | **0** | 通用 |
| 低 | **1** | 跟中弘 V1.5 不同 (V1.5 是 4) |
| 中 | **2** | 通用 |
| 高 | **3** | 跟中弘 V1.5 不同 (V1.5 是 1) |

代码已按此编码, 但调试时**别套用其他厂家的风速编码表**.

### 5.4 室温读取

寄存器偏移 4 (例如 n=0 内机, 读 4 寄存器即 reg 4) 是**有符号**:
- 0x00-0x32 → 0 ~ 50℃ (正)
- 0xce-0xff → -50 ~ -1℃ (补码)
- 0x33-0xcd → 非法 (适配器返回 undefined)

## 7. 常见故障排查

| 现象 | 可能原因 | 处置 |
| --- | --- | --- |
| 后台 `/api/health/devices` HVAC offline | 网关 IP 不对 / 网线松 / 网关没上电 | `ping 192.168.50.51` / `ping 192.168.50.52`, 检查交换机端口 |
| ping 通但 Modbus 502 不通 | 网关协议没设成 MBT / 防火墙挡 | Web 页面确认协议选 "MODBUS-TCP" |
| Modbus Poll 读 2000 等于 0 | 网关还没识别到空调 | 等 1-2 分钟 (网关 3 秒/次轮询搜索); 还不行检查 XYE 接线 |
| 2002 = 0 (识别 0 台内机) | XYE 极性反 / 内机未拨码 / 外机未上电 | 量 X/Y 极性; 找空调厂家用 ACBT 软件复读地址 |
| 某台内机控制不响应 | 该台地址拨错 / 通讯不通 | 在网关 Web 页 "内机列表" 查在线状态 |
| 1F 整组失联 但 2F 正常 | 1F 网关单独故障 | 重启 1F 网关; 换网线/适配器 |
| 1F + 2F 都失联 | 交换机问题 / 主控机网卡 | 主控机重启网卡; 排查交换机 |
| 室温显示乱码 (如 220℃) | 寄存器值进 0x33-0xcd 非法范围 | 适配器已过滤, 显示空; 找空调厂家排查温感 |
| Modbus 异常码 0x02 | 寄存器地址越界 (n > 该网关内机数 - 1) | 核对该网关的 n 范围 (1F 0-9, 2F 0-11) |
| 重启网关后内机数变 0 | 网关重新搜索中 | 等 1-2 分钟 |

## 9. 维护要点

- **XYE 总线接地**: 屏蔽层只在**外机一端**接地, 严禁两端接地 (引入接地环路, 干扰丢包)
- **网关供电**: 12V 适配器跟内机/外机分开走 220V 回路, 防压缩机启停冲击网关
- **冷媒侧故障**: 智能化不负责诊断, 故障代码 (寄存器偏移 5) 转空调厂家
- **冬季制热**: 商用 VRF 制热效果随外温下降, -5℃ 以下需开启电辅助 (线控器面板)
- **节能策略 (可选)**:
  - 闭馆模式: 14 个 hvac-zone 全部 turnOff
  - 夜间巡检: 维持 18℃ 防潮
  - 高温日预冷: 早 9 点提前 1 小时开馆区域空调

## 10. 现场快速参考

```
1F 网关:        192.168.50.51:502
2F 网关:        192.168.50.52:502
Modbus 协议:    MODBUS-TCP (MBT) V3.2
Modbus slaveId: 1 (默认)
品牌码:          15 (奥克斯)

寻址 (每网关独立 n=0..63):
  状态寄存器:    6n               (FC 0x03 读 6 寄存器)
  控制寄存器:    4000 + 4n        (FC 0x06 写 1 / FC 0x10 写 4)
  全部内机控制:  5000-5003        (该网关所有内机)
  网关性能信息:  2000-2005        (品牌/类型/内机数/温度上下限)

寄存器偏移 (内机):
  状态:
    +0  开关       0=关 / 1=开
    +1  模式       1=制热 / 2=制冷 / 4=送风 / 8=除湿
    +2  温度设定   直接十六进制 = 摄氏值
    +3  风速       0=自动 / 1=低 / 2=中 / 3=高
    +4  室温       有符号: 0x00-0x32=0-50℃, 0xce-0xff=-50~-1℃
    +5  故障代码   0=无故障
  控制 (前 4 项跟状态同):
    +0  开关       0=关 / 1=开
    +1  模式       1/2/4/8
    +2  温度       0x10-0x20 = 16-32℃
    +3  风速       0/1/2/3
```
