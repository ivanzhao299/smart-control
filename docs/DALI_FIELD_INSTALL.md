## 1. 工具准备

**必备**：

- 万用表（量 DC / AC / RS-485 差分电压）
- 网线测试仪 + 备用 Cat5e 网线 ×2
- USB-RS485 转换头（接调试串口用）
- 笔记本电脑（Windows，能跑 DALI 调试软件）
- DALI 调试软件（向厂家索取，元创智控自带 / 或 LunaCommander 通用版）
- 一字 + 十字螺丝刀（端子 / 拨码用）
- 标签机或绝缘胶带 + 油性笔（标线 / 标设备）

**推荐**：

- 备用 DC 12V/1A 适配器（万一原装坏）
- DALI 总线 RVVP 2×0.5mm² 双绞线（≥ 100m 备料）
- RS-485 屏蔽双绞线 RVSP 2×0.5mm² + 接地线 1×0.5mm²
- DIN 导轨 + 电箱压线端子若干

## 3. 物理安装

### 3.1 ARK-1220L 中控主机

- **位置**：1F 主控室 / 弱电机柜（带空调或通风）
- **网络**：固定 IP `192.168.50.10`，接弱电机柜的交换机
- **远程**：开 Tailscale + RDP，便于远程运维（参考 [DEPLOY_WINDOWS.md](../DEPLOY_WINDOWS.md)）

### 3.2 USR-TCP232-410s 转换器

- **位置**：1F 弱电机柜内 DIN 导轨上（紧挨 CY-DALI64A）
- **供电**：DC 12V/1A 适配器 → 顶部 + - 端子（认准极性，反接可能烧）
- **网口**：网线接弱电机柜交换机
- **RS-485 端**：底部 3 端子 A(+) / B(−) / G，向下走线到 CY-DALI64A

### 3.3 CY-DALI64A 灯光网关

- **位置**：1F 弱电机柜内 DIN 导轨（紧挨转换器）
- **供电**：AC 110-230V，接弱电机柜专用空开
- **拨码盘**：8 位拨码，**只用前 6 位**设 Modbus 地址，后 2 位不动
  - 单台部署 = 地址 1 = `10000000`（位 1 ON，其他 OFF）
- **2 个 RS-485 口**：
  - **控制串口**（label 通常在顶部接线端区）→ 接 USR-TCP232-410s
  - **调试串口**（不同端子组）→ 现场只在调试时接 PC，部署完拔掉
- **DALI 端子**（底部 DA1+ / DA2-）：两组并联输出，**接哪一组都行**，走 DALI 总线到 5 台 DA4-D
- **指示灯**：
  - `POWER` 常亮 = 电源 OK
  - `DALI` 通信时闪烁 = 总线在工作

### 3.4 DA4-D 调光器（× 5 台）

按现场分区分别装在对应电箱：

| 编号 | 安装位置 | 通道模式 | DALI 地址 | 接哪几路灯 |
| --- | --- | --- | --- | --- |
| DA4D-01 | 1F 公共电箱 F101 | **4CH** | 1, 2, 3, 4 | 前厅灯带 / 筒灯射灯 / 园区展示 / 走廊灯带 |
| DA4D-02 | 1F 公共电箱 F101 | **4CH** | 5, 6, 7, 8 | 走廊筒灯 / 吸顶灯 / 格栅长条 / 重点照明 |
| DA4D-03 | 1F F102 电箱 | **4CH** | 9, 10, 11, 12 | F102 灯带 / F102 预留 / F103 筒灯射灯 / F104 灯带 |
| DA4D-04 | 2F 公共电箱 F201 | **4CH** | 13, 14, 15, 16 | 2F 前厅 / 2F 走廊灯带 / 服务中心 / 配房 |
| DA4D-05 | 2F F202 电箱 | **3CH** | 17, 18, 19 | F202 办公 / F203 研究 / F204 指挥 |

**每台 DA4-D 接线**（4 个端子）：

```
顶部:
  + DC12-24V    →  恒压电源 +
  - DC12-24V    →  恒压电源 -
  DA1 / DA2     →  并接 CY-DALI64A 的 DALI 总线

底部 4 路 PWM 输出 (LED OUT 1/2/3/4):
  L+            →  各路 LED 灯带 +（共阳）
  L-1 / L-2 / L-3 / L-4 → 4 路灯带各自的 -
```

> 注意 LED 灯带必须是恒压（12V 或 24V）类型，对应 DA4-D 的恒压输出。

### 3.5 RS-485 接线（USR-TCP232-410s ↔ CY-DALI64A）

| USR-TCP232-410s | CY-DALI64A 控制串口 | 备注 |
| --- | --- | --- |
| `A(+)` | `A+`（或标 `D+`） | 信号正 |
| `B(−)` | `B-`（或标 `D-`） | 信号负 |
| `G`    | `GND`（或标 `G`） | 共地 |

**关键**：

- 用屏蔽双绞线（RVSP 2×0.5mm² + 1 根地线），线长 ≤ 100m
- 屏蔽层只在转换器一端接地
- **千万不要接到 CY-DALI64A 的"调试串口"**（位置不同，看 PDF 第 1 页接口示意图）

### 3.6 DALI 总线接线（CY-DALI64A ↔ 5 台 DA4-D）

```
CY-DALI64A: DA1+ ──┬─── DA4-D #1 DALI+ ─── DA4-D #2 ─── ... ─── DA4-D #5
            DA2-/DA1- ─┴─── DA4-D #1 DALI- ─── DA4-D #2 ─── ... ─── DA4-D #5
```

- DALI **无极性**，DA1/DA2 互换也工作
- 总线总长 ≤ 300m
- 用普通 RVV 2×0.75mm² 即可（DALI 协议自带电气隔离）
- 各 DA4-D 在总线上**并联**，没有主从顺序

## 4. 软件配置

### 4.1 USR-TCP232-410s（一次性）

**步骤**：

1. PC 用网线临时**直连**转换器网口（不经交换机）
2. PC 网卡设静态 IP `192.168.0.10`，掩码 `255.255.255.0`
3. 浏览器打开 **`http://192.168.0.7`**（出厂默认）
4. 账号密码 `admin / admin`

**配置参数**：

| 设置项 | 值 |
| --- | --- |
| 网络 → IP 模式 | 静态 |
| 网络 → IP 地址 | `192.168.50.20` |
| 网络 → 子网掩码 | `255.255.255.0` |
| 网络 → 网关 | `192.168.50.1` |
| 串口 → 工作模式 | **TCP Server** |
| 串口 → 本地端口 | **502** |
| 串口 → 波特率 | **9600** |
| 串口 → 数据 / 校验 / 停止 | **8 / NONE / 1** |
| 串口 → 流控 | NONE |
| 串口 → 打包时间 | **100ms**（影响 Modbus 帧分割） |
| 串口 → 打包长度 | 1024 |

点 **保存 + 重启**。重启后 PC 改回正常网段，浏览器再开 `http://192.168.50.20` 确认改成功。

### 4.2 CY-DALI64A 网关

**拨码盘**（侧面 8 位 DIP）：

| 拨码 1 (左) | 拨码 2 | 拨码 3 | 拨码 4 | 拨码 5 | 拨码 6 | 拨码 7 | 拨码 8 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| **ON** | OFF | OFF | OFF | OFF | OFF | × | × |

**= 地址 1**（二进制 `000001`，低位在左侧）。后 2 位不用。

### 4.3 DA4-D 通道模式

每台 DA4-D 上电后，**按 M 键 2 秒**进入 DALI 地址数量设置：

- ◀ ▶ 键切换 1CH / 2CH / 3CH / **4CH**
- 数显屏会显示当前模式
- 按 M 键或等 10 秒退出保存

**前 4 台设 4CH，第 5 台 (DA4D-05) 设 3CH**。

通道模式决定占用 DALI 地址数量（见 §3.4 表）。

### 4.4 DALI 驱动器地址搜索

1. 笔记本接 **USB-RS485 转换头** → CY-DALI64A 的**调试串口**（不是控制串口！）
2. 跑厂家 DALI 上位机软件（联系元创智控售后获取，文件名通常含 "DALI-Configurator" 或类似）
3. 软件里设串口波特率 9600
4. 点 **"搜索驱动器"** / **"Scan"**
5. 应该搜到 5 台 DA4-D 占用的短地址 1-19
6. **逐个测试单灯**：选中某个短地址 → 点 "Identify"，对应灯具会闪烁，确认对应物理回路
7. 用油性笔在 DA4-D 上贴标签：`DA4D-01 / addr 1-4`、`DA4D-02 / addr 5-8` ...

### 4.5 DALI 组分配

按现场分区把 19 个短地址加入 **DALI 组 1-12**：

| 组 | 描述 | 短地址成员（参考 §3.4） |
| --- | --- | --- |
| 1 | 1F 前厅 / 园区展示 | 1, 2, 3 (DA4D-01 ch1-3) |
| 2 | 1F 路演 / 洽谈区 | (按现场图纸分配, 可与组 1 重叠) |
| 3 | 1F 走廊 | 4, 5 (DA4D-01 ch4 + DA4D-02 ch1) |
| 4 | 1F 重点照明 / 灯箱 | 6, 7, 8 (DA4D-02 ch2-4) |
| 5 | 1F F102 企业展位 | 9, 10 (DA4D-03 ch1-2) |
| 6 | 1F F103 综合展销 | 11 (DA4D-03 ch3) |
| 7 | 1F F104 物贸交易 | 12 (DA4D-03 ch4) |
| 8 | 2F 前厅 / 走廊 | 13, 14 (DA4D-04 ch1-2) |
| 9 | 2F 企业服务中心 | 15, 16 (DA4D-04 ch3-4) |
| 10 | 2F F202 共享办公 | 17 (DA4D-05 ch1) |
| 11 | 2F F203 产业研究 | 18 (DA4D-05 ch2) |
| 12 | 2F F204 运营指挥 | 19 (DA4D-05 ch3) |

> 一个短地址可同时属于**多个**组。比如 "全开" 操作可以建一个组 13 = 1-19 所有地址，一键全开馆。

**保存到网关**：DALI 软件里点 "Save" / "Apply"，等几秒写入完成。

### 4.6 中控主机 (.env) 配置

打开 `D:\smart-control\backend\.env`，确认/修改：

```ini
# === DALI 灯光（核心配置）===
LIGHTING_ADAPTER_KIND=cy-dali64a
DALI_RTU_HOST=192.168.50.20
DALI_RTU_PORT=502
DALI_RTU_SLAVE_ID=1
DALI_RTU_FRAME_INTERVAL_MS=200
DALI_RTU_DEFAULT_FADE_SEC=0.7

# === 现场上线（必改）===
MOCK_MODE=false                # 关掉模拟, 走真实硬件
HOST_MACHINE=Advantech ARK-1220L-S6A2
PLATFORM=windows
```

保存后 PowerShell：

```powershell

pm2 logs smart-control-backend --lines 50 --nostream
# 期望看到: CyDali64aAdapter ready (host=192.168.50.20:502 slaveId=1)
```


**Step 3 — 后端探活**

```powershell
curl -X POST http://localhost:3000/api/system/runtime/health/probe
```

### 5.2 控制层

打开 **平板 https://cnjinhu.top/control/** 或本机 `http://localhost/`：

1. **侧栏 → 灯光** → 看到 12 个区域卡片
2. 任选一个区域（例 "一层前厅 / 园区展示"）：
   - 点 **「开启」** 按钮 → 现场对应灯具应该亮起
   - 调亮度滑块到 50% → 灯具应该平滑过渡到 50%
   - 点 **「关闭」** → 灯具熄灭
3. 全部 12 个区域逐个测试

后台 **/admin/scenes → 测试**：

## 6. 故障排查表

| 现象 | 检查 | 解决 |
| --- | --- | --- |
| `ping 192.168.50.20` 不通 | 网线是否插好；转换器 POWER 灯亮吗 | 重插网线 / 换网线 / 重启转换器 |
| 同上 | 网段是否对（主机 50.10 / 转换器 50.20） | 浏览器开转换器 web 改 IP |
| ping 通但 Test-NetConnection 失败 | 转换器是不是 TCP Server 模式 | 浏览器配置 → 工作模式 = TCP Server, 端口 502 |
| TCP 通但后端 state=offline | 波特率不是 9600 | 浏览器配置 → 串口 9600/8/N/1 |
| 同上 | RS-485 A/B 接反 / G 没接 | 万用表测 A-B 差分: 应 0.2V 左右; 互换 A B 试试 |
| 同上 | CY-DALI64A 拨码盘 slave_id 不是 1 | 检查拨码盘 + .env `DALI_RTU_SLAVE_ID` 一致 |
| 后端通了但灯不动 | DA4-D 数显屏地址显示什么 | 不显示就是没接 DALI 总线 / DA4-D 没通电 |
| 同上 | DALI 软件搜索能搜到驱动器吗 | 搜不到 = 总线断 / DA4-D 故障; 搜到 = 没分组 |
| 部分灯不亮 | DA4-D 通道模式拨码对吗（4CH / 3CH） | 长按 M 键重设 |
| 部分灯不亮 | LED 灯带 +/- 接反 | 灯带本身有方向, 反接不发光 |
| 灯一直最亮 / 调不了 | 灯具是否 DALI 类型，不是的话只能开关 | 换 DALI 调光灯具，或加 DALI-0/10V 转换器 |
| 调光闪烁 / 频闪 | DA4-D 频率 500Hz 与灯具不兼容 | 换更高 PWM 频率的调光器，或选无频闪灯具 |
| 多灯同时控制响应慢 | 帧间隔太短 | .env `DALI_RTU_FRAME_INTERVAL_MS` 加大到 300-500ms |

## 7. 安全注意

- **强电与弱电分开布线**：CY-DALI64A 的 AC 220V 跟 RS-485 / DALI 信号线**不要走同一根护管**，至少分两个管道，间距 ≥ 20cm
- **DALI 总线极性虽然无关，但施工时还是按 +/- 标识接**，便于后续维护
- **机柜接地**：弱电机柜必须接保护地（PE），RS-485 屏蔽层只在转换器一端接 PE
- **强电改线时切总闸**：CY-DALI64A 内部 DALI 电源是 16V 安全低压，但 220V 进线段必须**断电操作**
- **静电防护**：DA4-D / CY-DALI64A 都是电子板，接线前手摸金属机柜放静电

## 10. 多台 CY-DALI64A 扩展方案

> 当前金湖单台 CY-DALI64A 已能覆盖 64 短地址 / 16 组，余量 45 个空位。
> 本节给**未来扩展**或**多展厅 / 多楼栋**部署时的参考。

### 10.1 何时需要多台网关

| 触发条件 | 数值 |
| --- | --- |
| 灯具回路数 > 64 个 DALI 驱动 | 64 个短地址用尽 |
| 物理总线 > 300m | DALI 协议长度上限 |
| 跨楼栋 / 跨电箱独立部署 | 强电相位不同，要电气隔离 |
| 控制分区 > 16 个 DALI 组 | 单网关 16 组上限 |
| DALI 总线 250mA 供电不够 | 每驱动 ~2mA, 约 100+ 驱动满载（但仍受 64 短地址限制） |

### 10.2 三种拓扑对比

| 方案 | 适用 | 优势 | 劣势 |
| --- | --- | --- | --- |
| **A. 共总线多网关** | 同一栋楼, 不同分区 | 只加 CY-DALI64A，省转换器 | 全部网关共用一根 RS-485，并发响应稍慢 |
| **B. 多转换器多网关** | 跨楼栋 / 跨园区 | 物理隔离好，故障互不影响 | 网络规划复杂 + 多套 IP/.env |
| **C. 单网关 + DALI 信号放大器** | 总线 > 300m 但驱动 ≤ 64 | 不增 slave_id 复杂度 | 不扩驱动数，应用场景窄 |

CY-DALI64A 文档第 4 页《多系统示意图》官方推荐 **方案 A**，性价比最高。

### 10.3 方案 A: 共总线多网关（推荐）

```
ARK-1220L 中控主机 (192.168.50.10)
       │ 以太网
       ▼
USR-TCP232-410s (192.168.50.20:502)
       │ RS-485 总线 (单一)
       │
       ├── CY-DALI64A #1 (slave_id=1)  ─ DALI 总线 #1 (独立) ─→ 1F 灯具 (64 短地址内)
       ├── CY-DALI64A #2 (slave_id=2)  ─ DALI 总线 #2 (独立) ─→ 2F 灯具 (64 短地址内)
       └── CY-DALI64A #3 (slave_id=3)  ─ DALI 总线 #3 (独立) ─→ 3F 灯具 (64 短地址内)

每台网关有自己独立的 16V/250mA DALI 总线电源, 互不干扰.
```

**RS-485 接线（菊花链 / Daisy Chain）**：

```
转换器 A+ ──┬── CY-DALI64A #1 A+ ──┬── #2 A+ ──┬── #3 A+ ────┐
            │                       │            │              │  120Ω 终端电阻
转换器 B- ──┴── CY-DALI64A #1 B- ──┴── #2 B- ──┴── #3 B- ────┤  接在最末端
                                                                │  A+ 和 B- 之间
转换器 G  ──── CY-DALI64A #1 G  ─── #2 G  ─── #3 G  ─────────┘
```

> **关键**：
> - **总线两端必须接 120Ω 终端电阻** （A+ 与 B- 之间），不然会反射干扰
> - 走线尽量**直线菊花链**，不要 T 字分叉
> - 单总线网关数 ≤ 10 台（理论 RS-485 是 32 个 unit load，但实际越多越慢）

### 10.4 方案 A 拨码盘设置

每台 CY-DALI64A 设**不同** slave_id（前 6 位二进制，低位在左）：

| slave_id | 拨码 1 | 拨码 2 | 拨码 3 | 拨码 4 | 拨码 5 | 拨码 6 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | **ON**  | OFF | OFF | OFF | OFF | OFF |
| 2 | OFF | **ON**  | OFF | OFF | OFF | OFF |
| 3 | **ON**  | **ON**  | OFF | OFF | OFF | OFF |
| 4 | OFF | OFF | **ON**  | OFF | OFF | OFF |
| 5 | **ON**  | OFF | **ON**  | OFF | OFF | OFF |
| ... | 标准二进制低位在左 | | | | | |
| 32 | OFF | OFF | OFF | OFF | OFF | **ON**  |

**禁止重复**，重复地址会导致总线冲突。

### 10.5 中控配置（多网关）

#### .env

只配**默认网关**：

```ini
DALI_RTU_HOST=192.168.50.20    # 同一台转换器
DALI_RTU_PORT=502
DALI_RTU_SLAVE_ID=1            # 默认网关 (回退用)
```

#### 设备 address JSON（每个 zone 指明属于哪个网关）

在后台 `/admin/devices` 编辑每个 light_* 设备的 `address` 字段：

```json
{"slaveId": 1, "group": 3}    // 1 号网关的组 3
{"slaveId": 2, "group": 7}    // 2 号网关的组 7
{"slaveId": 3, "group": 1}    // 3 号网关的组 1
```

或者按物理楼层批量分配规则：

| 楼层 | slave_id | 组范围 | 设备命名 |
| --- | --- | --- | --- |
| 1F | 1 | 1-16 | `light_1f_*` |
| 2F | 2 | 1-16 | `light_2f_*` |
| 3F | 3 | 1-16 | `light_3f_*` |

> **当前限制**：平板 LightingPage 的 `setZoneBrightness(zoneId)` 直接调默认 slave_id 的组。多网关启用后**需扩展前端 zone 模型**，把 `zoneId` 改成 `{slaveId, group}` 对象，或建立全局 zoneId → (slaveId, group) 的映射表。属于一次性后端代码增强，估时 1-2 小时。

### 10.6 方案 B: 多转换器（跨楼栋）

适合：园区跨多栋建筑（A 馆 / B 馆），各楼栋有独立机柜，**不希望跨楼栋有长串总线**。

```
ARK-1220L
  ├── USR-TCP232-410s #1 (192.168.50.20) → A 馆 CY-DALI64A (slave_id=1)
  ├── USR-TCP232-410s #2 (192.168.50.21) → B 馆 CY-DALI64A (slave_id=1)
  └── USR-TCP232-410s #3 (192.168.50.22) → C 馆 CY-DALI64A (slave_id=1)
```

每个 CY-DALI64A 可以都设 `slave_id=1`（因为各自走独立的 TCP 连接），通过转换器 IP 区分。

后端需要支持**多 DALI host** — 当前 `cy-dali64a.adapter.ts` 只读 1 个 host，要改成 host 池或在 device.address 里放 host：

```json
{"host": "192.168.50.20", "slaveId": 1, "group": 1}    // A 馆
{"host": "192.168.50.21", "slaveId": 1, "group": 1}    // B 馆
```

> 这是较大的代码改造，需要时再做。

### 10.7 方案 C: 信号放大器（不扩短地址）

仅适合：DALI 总线物理长度 > 300m，但驱动数仍 ≤ 64 的特殊场景（比如长展厅，灯具沿一字排开）。

加 **DALI Bus Repeater** （元创智控等厂家都有）：

```
CY-DALI64A DA1+/DA2- ─── DALI 段 1 (前 150m, 32 驱动)
                    │
                    └─ DALI Repeater ─── DALI 段 2 (再 150m, 32 驱动)
```

驱动器还是统一 1-64 编号，软件层无感知。

### 10.8 扩展施工要点

1. **新加网关上电顺序**：先断 RS-485，再插 220V 电源，等 30s POWER 灯稳定，再连回 RS-485
2. **DALI 调试软件分别配置**：每台 CY-DALI64A 通过自己的**调试串口**单独搜索驱动 + 分组
3. **避免短地址冲突**：每台网关的 DALI 总线独立，所以**短地址 1-64 各自独立**，不需要全局唯一
4. **DALI 组号也独立**：1 号网关的"组 3"和 2 号网关的"组 3"是两组不同的驱动，互不影响
5. **总线供电**：每加一台 CY-DALI64A 都新增独立 16V/250mA 电源，DALI 端无需外接电源
6. **联调时分网关验收**：先 slave_id=1 测通，再加 slave_id=2，逐台叠加，故障好定位

### 10.9 性能与限制

| 项 | 单网关 | 3 网关共总线 | 10 网关共总线 |
| --- | --- | --- | --- |
| 单灯响应延迟 | 200-300ms | 200-300ms | 300-500ms |
| 全部并发刷新延迟 | < 1s | 1-2s | 3-5s |
| 推荐总线长度 | RS-485 < 1200m | < 1000m | < 800m |
| 推荐 .env 帧间隔 | `200ms` | `300ms` | `500ms` |

如果场景执行明显变慢，加大 `DALI_RTU_FRAME_INTERVAL_MS`：

```ini
DALI_RTU_FRAME_INTERVAL_MS=300   # 多网关时建议
```

## 11. 应急回退

如果现场设备调试遇到无法解决的问题，可以**临时切回 MOCK 模式**让前端可演示：

```powershell
# 编辑 D:\smart-control\backend\.env
# 改一行:
MOCK_MODE=true

# 重启
pm2 reload smart-control-backend --update-env
```

此时所有灯光 / LED / 音响 / 空调操作都走内存模拟，**不会**真实下发到硬件，但页面正常工作可以做演示 / UAT。等硬件修好后改回 `MOCK_MODE=false` 即可。
